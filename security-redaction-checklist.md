# Security Redaction Checklist

## 목적

공개 repo에 올라가면 안 되는 항목을 배포 전 확인한다.
이 체크리스트는 문서 작성 전, 코드 근거 추가 전, 발표자료 링크 추가 전에 모두 확인한다.

## 공개 전 제거해야 할 항목

- API Key
- Secret
- PlayFab Secret Key
- Firebase 설정
- `.env`
- 개인정보
- 팀원 개인정보
- 유료 에셋 원본
- Unity `Library` 폴더
- Unity `Temp` 폴더
- Unity `Obj` 폴더
- Unity `Build` 폴더
- Unity `Logs` 폴더
- 팀 프로젝트 전체 원본 코드
- 자소서 원문 전체
- 이력서 원본 전체

## 공개 전 판단 기준

| 분류 | 공개 가능 기준 | 보류 기준 |
| --- | --- | --- |
| 직접 구현 | 본인 작성 범위가 분리되고 민감정보가 제거된 코드/요약 | 원본 전체 코드, 비밀값, 팀 권한 미확인 자료 |
| 팀 결과물 / 협업 기여 | 본인 담당 범위와 팀 산출물의 경계가 명확한 요약 | 팀 전체 성과를 개인 성과로 보이게 하는 자료 |
| 확인 필요 | TODO로만 기록하고 원본은 Drive에 보관 | 출처, 권한, 공개 가능 여부가 불명확한 자료 |
| 과장 위험 | 금지 표현을 제거한 제한적 설명 | 검증되지 않은 성과, 전체 담당처럼 보이는 표현 |

## 확인 필요

- TODO: 공개 전 파일명 기준 민감정보 재검사.
- TODO: 공개 전 파일 내용 기준 키워드 재검사.
- TODO: Drive 링크 자체가 공개 가능한지 확인.
- TODO: 팀 프로젝트 자료는 팀원 개인정보와 유료 에셋 포함 여부 확인.

## 과장 위험 / 금지 표현

- 공개 가능한 근거가 없는 구현을 직접 구현으로 표현하지 않는다.
- 팀 전체 결과물을 개인 단독 구현처럼 표현하지 않는다.
- 검증하지 않은 실행 결과를 완료된 검증처럼 표현하지 않는다.
- 전부 직접 구현
- 전체 시스템 설계
- 서버 전체 구현
- 라이브 서비스 운영
- 상용 수준 완성
- 모든 버그 해결
- 팀 전체 코드 주도
- QA 전체 담당
- 밸런스 전체 확정

## 허용 가능한 표현 방향

- 담당 범위 내 구현
- 특정 기능의 구조 개선
- 검증 로그 기반 문제 추적
- 팀 결과물 중 본인 기여 범위 명시
- 코드/문서/테스트 근거 기반 개선
- 포트폴리오 보조 근거
