# Self Introduction Materials

## 목적

자소서에 사용할 소재를 프로젝트 근거와 연결해 정리한다.
감정적 서사는 나중에 작성하되, 먼저 어떤 문제를 어떻게 만들고 어떻게 해결했는지
근거 중심으로 분리한다.

## 현재 확정된 사실

- 자소서 완성본을 작성하기 전 단계의 소재 정리 문서다.
- 구현 사실이 확인되지 않은 내용은 TODO 또는 확인 필요로 남긴다.
- 프로젝트 경험은 직접 구현, 팀 결과물 / 협업 기여, 확인 필요, 과장 위험으로 분리한다.
- 자소서 소재는 `projects/`의 claim 정리와 `portfolio/`의 공개 가능 근거를 대조한 뒤 작성한다.
- 같은 프로젝트 설명을 이 문서에 장문으로 반복하지 않고 링크와 요약만 둔다.

## 참조 우선순위

1. `career/claim-boundary.md`에서 말할 수 있는 범위를 확인한다.
2. `projects/<project>/`에서 자소서/면접용 claim 후보와 확인 필요 항목을 확인한다.
3. `portfolio/<project>/`에서 공개 가능한 정제 설명과 sanitized evidence를 확인한다.
4. `drive-links.md`에서 원본 자료 위치를 확인하되 원본을 GitHub로 복사하지 않는다.

## 과장 위험 / 금지 표현

- 감정적 표현만 있고 구현 근거가 없는 문장은 보류한다.
- 실제 해결하지 않은 문제를 해결했다고 쓰지 않는다.
- 전부 직접 구현, 전체 시스템 설계, 모든 버그 해결처럼 범위가 큰 표현은 근거가 생기기 전까지 금지한다.
- Project3를 완성 게임 포트폴리오의 메인 성과처럼 쓰지 않는다.
