이 문서는 GitHub 기준 문서와 Drive 제출 산출물의 연결을 정리합니다.

최종 검토 기준은 다음과 같습니다.

- 기술문서: Drive `external/submission-technical-document-external-v2.pdf`, `external/submission-technical-document-external-v2.docx`
- 자소서: GitHub `career/final-self-introduction.md`의 `## 1. 제출용 자소서` 본문
- PPT: Drive `ppt/unity-client-bootcamp-projects-v2.pptx`

GitHub에는 Markdown 기준 문서만 둡니다. DOCX, PDF, PPTX, 이미지, 영상, 압축파일은 Drive 보관 대상으로 둡니다.

## 2. 외부 제출 / 강사님 검토 후보

### 기술문서

### PPT

- Drive `ppt/unity-client-bootcamp-projects-v2.pptx`

사용 기준:

- 기술문서 v2와 자소서 피드백 반영본의 핵심 문장에 맞춘 검토용 PPTX입니다.
- 핵심 키워드는 `상태 변경`, `실패 복구`, `검증 경계`입니다.

### 자소서

- GitHub `career/final-self-introduction.md`

사용 기준:

- 실제 제출 시에는 Markdown 전체가 아니라 `## 1. 제출용 자소서` 본문만 복사합니다.
- 검토 메모, 글자 수, 사용 소재, 제한 표현 메모는 제출용 본문이 아니라 내부 확인용입니다.
- 현재 본문은 공백 포함 1925자, 공백 제외 1508자 기준입니다.

## 6. 제출 전 사용 순서

1. Drive `external/submission-technical-document-external-v2.pdf` 또는 `.docx`를 기술문서 제출 후보로 확인합니다.
2. GitHub `career/final-self-introduction.md`의 `## 1. 제출용 자소서` 본문만 복사해 자소서 제출 후보로 사용합니다.
3. Drive `ppt/unity-client-bootcamp-projects-v2.pptx`를 PPT 검토본으로 사용합니다.
4. 표현이 애매한 항목은 `career/claim-boundary.md`와 `career/verification-needed.md`로 다시 확인합니다.
5. 기업 제출 전 Drive 원본 자료, 팀 공개 승인, 민감정보 제거 여부를 확인합니다.

## 7. 주의 사항

- `internal-review/` 문서는 외부 기업 제출용으로 사용하지 않습니다.
- `draft.pptx`는 과거 초안이며, 최종 검토 기준은 `unity-client-bootcamp-projects-v2.pptx`입니다.
- 자소서는 Markdown 전체가 아니라 본문만 제출합니다.
- GitHub에는 DOCX, PDF, PPTX, 이미지, 영상, 압축파일을 올리지 않습니다.
- Drive 원본 파일을 GitHub로 복사하지 않습니다.
- 확인되지 않은 구현 사실을 직접 구현으로 확정하지 않습니다.
