# Project 3 - UI Art Pipeline Evidence

## 목적

Project3 UI/아트 파이프라인 관련 근거를 정리한다.
현재 repo 기준으로는 전체 UI 아트 제작이 아니라 UI 리소스 기준, 지시문, 검수, Unity 적용 여부를 분리하는 보조 근거로 다룬다.

## 현재 확정된 사실

- 현재 repo에는 UI/아트 원본 이미지, Unity prefab, sprite 설정, 실제 적용 로그가 없다.
- `portfolio/03-project3-fisher/README.md`와 `ai-assisted-workflow.md`는 AI/Codex 기반 작업 운영 흐름을 보조 근거로 둔다.

## 직접 구현 근거

- 현재 repo 기준 UI 적용, 에셋 연결, sprite import 설정을 직접 구현으로 확정할 근거는 없다.
- 지시문 작성, 기준 분리, 검수 흐름은 추가 자료가 있으면 보조 소재로 사용할 수 있다.

## 팀 결과물 / 협업 근거

- UI/아트 리소스는 팀원 작업물, AI 산출물, 본인 지시/검수/적용 범위를 분리해야 한다.
- AI 이미지 생성 결과를 본인 직접 제작 그래픽처럼 표현하지 않는다.
- Unity 적용 여부와 지시문 작성/검수 여부를 분리한다.

## 확인 필요

- UI 리소스/아이콘/스프라이트/9-slice 관련 실제 자료.
- transparent background, padding, naming convention 기준 문서.
- sprite repair 또는 icon generation prompt.
- Codex 적용 지시문과 GPT handoff 자료.
- Unity 적용 여부, prefab/scene 연결, 검수 로그.
- 유료 에셋 원본, 팀원 작업물, 이미지 원본 공개 가능 여부.

## 자소서 활용 가능 포인트

- 안전한 표현:
  - UI 리소스 적용 전 기준, 명명 규칙, 여백, 투명 배경 문제를 정리했다.
  - AI 이미지 생성 결과를 그대로 본인 제작물로 말하지 않고, 적용 기준과 검수 지시문으로 분리했다.
  - Unity 적용 여부와 지시문 작성/검수 여부를 분리했다.

## 면접 예상 질문

- UI 리소스 기준을 코드/Unity 적용 전 어떻게 정리했는가?
- AI 산출물, 팀원 아트, 본인 검수 범위를 어떻게 구분했는가?
- Unity 적용 완료 여부는 어떤 근거로 확인할 수 있는가?

## 과장 위험 / 금지 표현

- 팀원 아트 작업을 본인 산출물처럼 표현하지 않는다.
- 유료 에셋 원본을 공개하지 않는다.
- 전체 UI 아트 제작.
- AI 이미지 산출물을 직접 제작 그래픽처럼 표현.
- Unity 전체 UI 적용 완료 단정.
- 아트 파이프라인 전체 주도.

## 필요한 추가 자료

- UI resource 기준 문서.
- sprite/icon prompt 또는 handoff 자료.
- Unity 적용 스크린샷 또는 prefab/scene 검증 로그.
- 공개 가능한 이미지/영상은 Drive 링크로 관리.
