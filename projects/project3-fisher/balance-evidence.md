# Project 3 - Balance Evidence

## 목적

Project3 밸런스 관련 근거를 정리한다.
현재 repo 기준으로는 밸런스 확정 성과가 아니라 workbook, validator, CSV/JSON 계약, 검증 문서화 후보를 분리하는 보조 근거로 다룬다.

## 현재 확정된 사실

- 사용자 제공 키워드: 밸런스.
- `portfolio/03-project3-fisher/tools-and-verification.md`에 balance validator와 economy simulator가 로컬 검토용 도구 후보로 정리되어 있다.
- `career/claim-boundary.md`와 `career/interview-keywords.md`는 전체 밸런스 확정을 금지 표현으로 분리한다.

## 직접 구현 근거

- 현재 repo 기준 직접 작성한 balance workbook, formula, validator 실행 로그는 없다.
- 직접 구현 claim으로 쓰려면 workbook 원본, 변경 이력, 실행 로그, 실제 반영 여부가 필요하다.

## 팀 결과물 / 협업 근거

- 밸런스 수치와 tuning 기준은 팀 결과물일 수 있으므로 개인 담당 범위를 분리해야 한다.
- CSV/JSON 계약, Google Sheet 또는 Excel authoring, pacing dashboard는 현재 후보로만 둔다.

## 확인 필요

- balance workbook 원본.
- validator 또는 economy simulator 실행 로그.
- CSV/JSON 계약 근거.
- pacing dashboard 또는 tuning evidence.
- 실제 런타임 반영 여부.
- 팀 기준 수치와 개인 검토/정리 범위.

## 자소서 활용 가능 포인트

- 안전한 표현:
  - 밸런스 수치 확정이 아니라 검증 기준과 근거 분리를 관리했다.
  - 밸런스 workbook과 검증 문서화를 취업 보조 근거로 정리했다.
  - 런타임 반영 여부는 확인 필요로 분리했다.

## 면접 예상 질문

- 밸런스 수치와 검증 로그의 출처는 무엇인가?
- workbook 후보와 실제 런타임 반영 상태를 어떻게 구분했는가?
- validator나 simulator 결과를 왜 최종 재미 판단이 아니라 risk warning으로 보는가?

## 과장 위험 / 금지 표현

- 문서상 후보 수치를 실제 반영 완료 수치처럼 표현하지 않는다.
- 전체 게임 밸런스 확정.
- 상용 밸런싱 완료.
- 모든 수치 검증 완료.
- 실제 런타임 반영 완료 단정.

## 필요한 추가 자료

- workbook 또는 sheet 링크.
- validator/simulator 실행 로그.
- 변경 전후 수치 비교표.
- 팀 기준 수치와 개인 작업 범위 메모.
